# Deref y DerefMut

`Deref` es un rasgo que permite utilizar `*` para desreferenciar una variable. `Deref` ha aparecido anteriormente cuando se usaba una estructura de tupla para crear un nuevo tipo. Ahora es el momento de aprender su uso.

Se conoce ya que una referencia no es lo mismo que un valor:

```rust
// ‚ö†Ô∏è
fn main() {
    let valor = 7; // Esto es un i32
    let referencia = &7; // Esto es un &i32
    println!("{}", valor == referencia);
}
```

En este caso, Rust no devuelve `false` en la comparaci√≥n. Da error de compilaci√≥n:

```text
error[E0277]: can't compare `{integer}` with `&{integer}`
 --> src\main.rs:4:26
  |
4 |     println!("{}", valor == referencia);
  |                          ^^ no implementation for `{integer} == &{integer}`
```

La soluci√≥n en este caso es el uso de `*`. Que dar√° como resultado `true`.

```rust

    let valor = 7;
    let referencia = &7;
    println!("{}", valor == *referencia);
}
```

A continuaci√≥n, se construye un tipo simple que solamente contiene un n√∫mero. Este tipo ser√° parecido al tipo `Box`. Si solamente se crea el tipo con el n√∫mero, no se podr√° hacer mucho con √©l.

No se puede usar `*`, como s√≠ se puede con `Box`:

```rust

    let mi_numero = GuardaUnNumero(20);
```

El error dice:

```text

 --> src/main.rs:6:20
  |
6 |     println!("{}", *mi_numero + 20);
  |                    ^^^^^^^^^^
```

S√≠ se podr√≠a hacer algo como esto `println!("{:?}", mi_numero.0 + 20);` , pero entonces solo se est√°n sumando un `u8` a `20`. Lo que ser√≠a pr√°ctico es sumar la variable con el valor. El mensaje `cannot be dereferenced` da una pista: es necesario implementar `Deref`. A los elementos simples que implementan este rasgo se los suele llamar *punteros inteligentes*- Un punterio inteligente puede apuntar a su elemento, tiene informaci√≥n adicional sobre √©l y puede tener y usar diversos m√©todos. El que se est√° construyendo puede hacer poca cosa en este momento, sumar un n√∫mero e imprimirlo en `println!` ya que implementa `Debug`.

Cabe destacar un hecho interesante: `String` es un puntero inteligente a `&str` y `Vec` es un puntero inteligente a un array u otro tipo. Desde el comienzo, se han estado presentando punteros inteligentes.

La implementaci√≥n de `Deref` no es dif√≠cil y los ejemplos de la librer√≠a est√°ndar son f√°ciles. (Este es un c√≥digo de ejemplo de la librer√≠a est√°ndar)[https://doc.rust-lang.org/std/ops/trait.Deref.html]:

```rust

struct DerefExample<T> {
    value: T
}

impl<T> Deref for DerefExample<T> {
    type Target = T;

    fn deref(&self) -> &Self::Target {
        &self.value
    }
}

fn main() {
    let x = DerefExample { value: 'a' };
    assert_eq!('a', *x);
}
```

Siguiendo el ejemplo, el tipo que se est√° creando necesita:

```rust

    type Target = u8; // Recuerda, este es el "tipo asociado": el tipo al que va unido.
                      // Hay que poner el tipo correcot de retorno para Target

    fn deref(&self) -> &Self::Target { // Rust llama a .deref() cuando se usa *. solo se ha definido el Target como u8
        &self.0   // Se elige &self.0 porque estamos en un struct tupla. En una struct con campos con nombres se usar√≠a "&self.numero"
    }

```

Ahora s√≠ se puede usar `*`:

```rust

    type Target = u8;

    fn deref(&self) -> &Self::Target {
        &self.0
    }

    let mi_numero = GuardaUnNumero(20);
    println!("{:?}", *mi_numero + 20);

```

El c√≥digo anterior imprime `40` sin que haya habido necesidad de referirse al valor almacenado en el `struct` con `mi_numero.0`. As√≠ se dispone de los m√©todos de `u8` y se pueden a√±adir otros m√©todos a `GuardaUnNumero`. A continuaci√≥n, se observa esto mediante el uso de un m√©todo denominado `.checked_sub()`. Se trata de una resta segura que devuelve un `Option`. Si puede hacer la resta, devuelve un `Some` con el resultado en su interior. Si no puede hacerla, devuelve `None`.  Se debe recordar que `u8` no puede guardar n√∫meros negativos. Por ello, es m√°s seguro usar `.checked_sub()` evitando que el programa entre en p√°nico.

```rust

    fn imprime_el_doble_del_numero(&self) {

    }

    println!("{:?}", mi_numero.checked_sub(100)); // Este m√©todo viene de u8
    mi_numero.imprime_el_doble_del_numero(); // Este viene del struct GuardaUnNumero

```

Este c√≥digo imprime:

```text
None

```

## DerefMut

Tambi√©n se puede implementar `DerefMut` que permite modificar los valores a trav√©s de `*`. Es muy parecido. Es necesario haber implementado `Deref` para poder implementar `DerefMut`.

```rust
use std::ops::{Deref, DerefMut};

struct GuardaUnNumero(u8);

impl GuardaUnNumero {
    fn imprime_el_doble_del_numero(&self) {
        println!("{}", self.0 * 2);
    }
}

impl Deref for GuardaUnNumero {
    type Target = u8;

    fn deref(&self) -> &Self::Target {
        &self.0
    }
}

impl DerefMut for GuardaUnNumero { // Aqu√≠ no se necesita Target = u8; ya que se conoce debido a Deref
    fn deref_mut(&mut self) -> &mut Self::Target { // es igual que en Deref, pero con mut
        &mut self.0
    }
}

fn main() {
    let mut mi_numero = GuardaUnNumero(20);
    *mi_numero = 30; // DerefMut permite que se pueda hacer esto
    println!("{:?}", mi_numero.checked_sub(100));
    mi_numero.imprime_el_doble_del_numero();
}
```

Esto imprime:

```rust
None
60
```

Se observa que `Deref` permite muchas posibilidades.

Por ello, la librer√≠a est√°ndar dice que *Con el fin de evitar confusiones, Deref solo deber√≠a implementarse para punteros inteligentes*. Esto se debe a que se pueden hacer cosas extra√±as con `Deref` si un tipo es compuesto. A continuaci√≥n se muestra un ejemplo desconcertante. Se crea un `struct Personaje` para un juego. Este personaje tiene diversos atributos.
